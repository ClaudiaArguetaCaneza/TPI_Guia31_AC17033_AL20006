CONCLUSIÓN

A partir de lo visto en la realización del ejemplo 1 y los ejercicios 1a y 1b se puede afirmar que los documentos XML son muy bien estructurados y ordenados que mediante parámetros como el abrir y cerrar etiquetas siempre hace que tenga esta estructura muy ordenada. Además, que debido a que no se siguió un DTD se puedo escribir la estructura y las etiquetas a nuestro gusto y así poder tener diferentes fichas con diferentes tipos de datos. El uso de la herramienta XML Copy Editor permite que se puedan verificar todo tipo de errores y también comprobar si el documento está bien formado y esto hace que la herramienta sea muy útil e intuitiva.

Tras la realización del ejemplo 2 y los ejercicios 2a y 2b puedo concluir con aspectos como que XML no permite el uso de espacios en sus etiquetas, sin embargo, en sus valores si que lo permite, lo cual es de esperarse debido a que debemos escribir todo tipo de atributos. Al momento de crear etiquetas es importante que se haga con un orden lógico para una mejor comprensión de datos y evitar la irracionalidad de las cosas como que la moneda no es de un solo departamento de un país sino de todo el país por lo tanto la etiqueta moneda no debe ir dentro de la etiqueta departamento.

Gracias a herramientas web como xml viewer podemos convertir de un lenguaje a otro (xml a json) en tan solo un par de clics. Esto importante debido a que un desarrollador puede necesitar pasar los datos a un lenguaje y así poder tener una enorme eficiencia en cualquier desarrollo web.

Tras el desarrollo de los archivos css puedo afirmar que es muy intuitivo al momento de querer dar un estilo a nuestros archivos xml, basta con definir una estructura css para cada una de las etiquetas donde podemos observar diferentes colores y tamaños que nosotros establezcamos para cada uno de nuestros archivos xml y de esa manera presentar los datos de una forma más creativa.
